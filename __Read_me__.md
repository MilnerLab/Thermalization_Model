# Centrifuge Experiment — Angulon Model

Numerical simulation of molecular alignment in superfluid helium droplets driven by a
centrifuge laser pulse (chirped rotating field). The angulon model treats the impurity
rotor coupled to the superfluid bath as a quantum polaron. The pipeline produces
theoretical predictions for the observable ⟨cos²θ₂D⟩(t) and compares them against
experimental centrifuge data for CS2 and OCS molecules.

---

## Experimental data (`Experimental_data/`)

### Raw data

Four CSV files, one per experimental dataset:

| File | Molecule | Drive |
|------|----------|-------|
| `CS2_accelerating_droplets.csv` | CS₂ | accelerating chirp |
| `CS2_decelerating_droplets.csv` | CS₂ | decelerating chirp |
| `OCS_accelerating_droplets.csv` | OCS | accelerating chirp |
| `OCS_decelerating_droplets.csv` | OCS | decelerating chirp |

Each CSV has three columns: `t` (s), `⟨cos²θ₂D⟩`, measurement error. The signal is
the 2D projection of the molecular alignment averaged over the droplet ensemble.

### Fitted drive parameters

The oscillation envelope in the raw signal is fitted (by `Plot_signal_fit_oscillations.py`,
standalone) to a chirped rotating model:

    Ω₀(t) = f₀ + ramp × t,    V(t) = V₀ × Gaussian(t)

The fitted rotational chirp parameters extracted from this fit, and used in the
theoretical model, are:

| Molecule | `f₀` (GHz) | `ramp` (MHz/ps) | `phi0` (rad) |
|----------|-----------|----------------|-------------|
| CS₂ | 18.997 | 92.08 | 0.0110 |
| OCS | 22.499 | 97.59 | 0.5720 |

(These numbers live in the `CS2` / `OCS` entries of `CASES` in `01_Parameters.py`.)

The amplitude scaling factor `alpha` (dimensionless) is not fitted from the oscillation
envelope but from the overall signal amplitude: it is determined by least-squares at the
plot stage (`plot_experimental_data.py`) for each dataset / model combination, and
reported in the pipeline output. Current values:

| Dataset | CS₂ model | CS₂ renorm. | OCS model | OCS renorm. |
|---------|-----------|-------------|-----------|-------------|
| CS₂ accelerating | 0.109 | 0.0675 | — | — |
| CS₂ decelerating | 0.102 | 0.0651 | — | — |
| OCS accelerating | — | — | 0.555 | 0.269 |
| OCS decelerating | — | — | 0.658 | 0.322 |

### PDF plots

For each dataset `{MOLECULE}_{accelerating|decelerating}_droplets` the following PDFs
are produced by `plot_experimental_data.py`:

- `_cos2theta2D_vs_t.pdf` — raw signal only
- `_cos2theta2D_vs_t_with_model.pdf` — signal + bare-rotor model overlay (scaled by alpha)
- `_cos2theta2D_vs_t_with_model_renormalised.pdf` — signal + renormalised model overlay
- `_cos2theta2D_vs_t_prediction.pdf` — model prediction only (bare rotor)
- `_cos2theta2D_vs_t_prediction_renormalised.pdf` — model prediction only (renormalised)
- `_cos2theta2D_vs_t_with_predictions.pdf` — signal with all model predictions overlaid
- `_frequency_map_vs_t.pdf` — continuous wavelet frequency map of the signal
- `_cos2_signal_fit_oscillations.pdf` (CS₂ and OCS accelerating only) — oscillation fit

### Compiled prediction file

`prediction_data.h5` is generated by `generate_prediction_data.py` (post-stage in the
pipeline). It gathers the model ⟨cos²θ₂D⟩(t) traces for all four cases (CS2, CS2_renormalised,
OCS, OCS_renormalised) and their metadata into a single HDF5 for the plotting stage.

## Computation scripts

### `01_Parameters.py` — central parameter hub

All parameters are defined here. Two-level structure:

- `BASE_DEFAULTS` — shared baseline for all cases (molecule-independent)
- `CASES` — per-case overrides (a dict of dicts)

`get_defaults_for_case(case_name)` merges `BASE_DEFAULTS` with the case overrides and
injects all derived path variables. Every computation script calls this at startup via
`importlib`.

**Defined cases:**

| Case | Molecule | Rotational model | Notes |
|------|----------|-----------------|-------|
| `Default` | generic | free rotor (1) | reference, no molecule-specific coupling |
| `CS2` | CS₂ | free rotor (1) | B=3.27 GHz, fitted drive params, CS₂–He coupling |
| `CS2_renormalised` | CS₂ | centrifugal distortion (2) | B★=0.73 GHz, D★=1.2×10⁻³ GHz |
| `OCS` | OCS | free rotor (1) | B=6.08 GHz, fitted drive params, OCS–He coupling |
| `OCS_renormalised` | OCS | centrifugal distortion (2) | B★=2.18 GHz, D★=9.5×10⁻³ GHz |

The `CS2` / `OCS` cases use the bare rotational spectrum E_J = B·J(J+1). The
`_renormalised` variants use a modified spectrum that saturates at high J, modelling the
effective (angulon-renormalised) rotational constant:

    E_J★ = B·J(J+1) − δB · (ΔB/D★) · J(J+1) / (ΔB/D★ + J(J+1))

controlled by `B_star` and `D_star`.

**Free parameters** (the two main physics knobs, not fixed by the experimental fit):

- `E0` (laser intensity proxy, V/m²): scales the centrifuge coupling strength
  V₀ = ½ Δα · E₀ · 6.32×10⁻¹⁰. Typical values 1×10¹⁰ (CS₂/OCS bare) to 3×10¹⁰
  (renormalised cases).
- `tau_steady_state` (ns): thermal relaxation time τ used to compute the initial
  density matrix before the pulse via a causal half-Gaussian average. Smaller τ means
  the rotor thermalises more fully to the instantaneous Hamiltonian before the pulse.
  Typical values 0.4 ns (CS₂) to 1.0 ns (OCS).

**How to change parameters:** edit `CASES` or `BASE_DEFAULTS` in `01_Parameters.py`.
The pipeline caches outputs by key parameter values; after any change it will
automatically detect stale stages and recompute only what is needed.

Environment variable overrides (useful for quick one-off tests without editing the file):
- `PENDULON_NT_MAIN` — override `Nt_main` (number of time steps in the main solve)
- `PENDULON_NT_PLOT_ROTATING_OBSERVABLES` — override `Nt_plot_rotating_observables`
- `PENDULON_NPROC` — override `nproc` (number of parallel workers)

### `01b_precompute_Ylm_blocks.py` — spherical harmonic matrices

Computes Y_{λ,μ}(θ,φ) matrix elements in the truncated |J,M⟩ basis (up to J_max) and
stores them to `data/01_spherical_harmonics/Ylm_blocks_JM.h5`. This file is **shared
across all cases** (depends only on J_max and λ_max, not on the molecule). It is almost
always cached and takes only seconds if recomputed.

---

### `01c_precompute_steady_state.py` — thermal initial state

Computes the exact thermal density matrix ρ_target(t) = exp[−β H_target(t)] / Z for the
driven Hamiltonian H_target(t) = B J(J+1) + V_drive(t), evaluated at each time step along
the time grid. This is the reference "perfectly thermalised" state used as a target for
the angulon dynamics. Per-case; cached by drive trajectory hash. Takes ~20 s per case.

---

### `02a_superfluid_helium.py` — bath library (not run directly)

Module defining the superfluid helium phonon bath: dispersion relation, density of states,
and molecule–bath interaction potentials U_λ(k) from the coupling constants
(C06, C08, C26, C28, r0, r2, …). Imported by `02b_bath_angulon.py`.

---

### `02b_bath_angulon.py` — angulon self-energy and spectrum

Computes the one-loop Fock self-energy Σ_λ(ω) and the angulon Green's function:

    G_λ(ω) = 1 / [ω − B λ(λ+1) − Σ_λ(ω)]

Extracts renormalised energy levels from the peaks of the spectral function A_λ(ω),
estimates B★ from the first level spacing, and shows the rotating-frame spectrum.
Always re-run (fast, ~6 s). Produces diagnostic PDFs in `figs/[CASE]/02_bath_and_angulon/`.

---

### `03a_free_rotor_drive_compute.py` — driven Hamiltonian eigensystem

Builds H(t) = B J(J+1) + V_drive(t) cos²φ sin²θ in the |J,M⟩ basis and diagonalizes it at
each of the `Nt_main` = 500 time steps. The resulting eigenvalues and eigenvectors are
stored to `data/[CASE]/03_free_rotor_drive/free_rotor_drive_diagonalization.h5`.
Cached by Nt_main and drive trajectory. Takes ~15–25 s per case.

---

### `03b_free_rotor_drive_observables.py` — observable matrices

Precomputes the matrices of all relevant angular observables (⟨cos²θ₂D⟩, sin²θ, x², y²,
…) in the antipodally symmetric |J,M⟩ basis. Per-case but rarely changes; cached.
Writes to `data/[CASE]/03_free_rotor_drive/observable_projections.h5`.

---

### `03c_free_rotor_drive_thermal.py` — thermal time evolution **(most expensive)**

Reads the 03a eigensystem and 03b observable matrices and computes the thermally-weighted
time evolution of all observables. The initial state is a mixed thermal state with
relaxation time `tau_steady_state`; the time evolution is unitary in the instantaneous
eigenbasis of H(t). Uses multiprocessing (up to `nproc` workers, automatically reduced
for memory). Writes `data/[CASE]/03_free_rotor_drive/observable_matrices.h5`.

This is the **dominant cost**: ~18–20 min per case on a typical workstation. Cached by
`Nt_plot_rotating_observables` and tau parameters; only reruns when those change.

---

### `03d_free_rotor_drive_plot.py` — driven system diagnostic plots

Loads the 03a eigensystem and produces diagnostic figures: bare and drive Hamiltonian
matrices, renormalised energy levels compared to bare rotor, eigenvector composition.
Writes PDFs to `figs/[CASE]/03_free_rotor_drive/` and `…/figs_technical/`.

---

### `generate_prediction_data.py` — compile predictions (post-stage)

Reads `observable_matrices.h5` for each of the four molecule cases and assembles
`Experimental_data/prediction_data.h5` with model traces and metadata. Fast (~1 s).

---

### `plot_experimental_data.py` — comparison plots (post-stage)

Reads experimental CSVs and `prediction_data.h5`, fits `alpha` by least squares,
computes continuous wavelet frequency maps, and produces all PDFs in `Experimental_data/`.
Takes ~5 s.

---

### `Plot_signal_fit_oscillations.py` — drive parameter extraction (standalone)

Not part of the pipeline. Fits the oscillation envelope in the raw CSV signal to a
chirped-Gaussian-decay model to extract f₀, ramp, phi0. Run manually to update the
drive parameters stored in `CASES`.

---

## Pipeline (`01_Pipeline.py`)

Orchestrates all stages across all enabled cases. Run with:

```bash
python 01_Pipeline.py
```

Key controls at the top of the file:

```python
ACTIVE_RUN_PLAN = "run_plan_full"   # which stages to run
CASE_ENABLED = {                     # toggle individual cases on/off
    "Default": True,
    "CS2": True,
    ...
}
CASE_STAGE_OVERRIDES = {}            # per-case stage overrides
```

Available run plans:

| Plan | Stages included |
|------|----------------|
| `run_plan_full` | all stages (02B → 01B → 01C → 03A → 03B → 03C → 03D) + post |
| `run_plan_03` | 01B, 01C, 03A, 03B, 03C, 03D (skip bath) |
| `run_plan_plot` | 03B, 03C, 03D only |
| `run_plan_02` | 02B only |
| `run_plan_custom` | 03C, 03D only |

The pipeline checks cached HDF5 outputs before each stage and skips anything already up
to date. Typical wall times for a full run from scratch (all 5 cases):

| Stage | Time per case |
|-------|--------------|
| 02B bath/angulon | ~6 s |
| 01C steady state | ~20 s |
| 03A drive eigensystem | ~15–25 s |
| 03B observables | ~15 s (if not cached) |
| **03C thermal evolution** | **~18–20 min** |
| 03D diagnostic plots | ~25 s |
| Post: generate + plot | ~6 s total |

**Total for a fresh full run: ~1 h 40 min** (5 cases × ~20 min for 03C dominates).

under `data/` is always shared (a single copy used by all cases).
